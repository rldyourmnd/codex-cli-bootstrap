## Governance Model

This repository uses an owner-led maintainership model.

- The owner defines project direction, release policy, branding, and final merge authority.
- Additional maintainers may be appointed by the owner.
- Contributors may propose code, docs, and workflow changes through pull requests.

## Maintainer Responsibilities

- Preserve runtime reproducibility
- Protect secret-handling and safety guarantees
- Keep documentation and module hierarchy current
- Reject changes that reduce portability, auditability, or operator clarity

## Decision Areas Reserved To The Owner

- License changes
- Branding and project naming
- Governance changes
- Baseline agent architecture changes
- Release and roadmap decisions

## Contribution Path

1. Open an issue or pull request.
2. Follow [`CONTRIBUTING.md`](CONTRIBUTING.md).
3. Pass required checks and documentation updates.
4. Wait for maintainer review and approval.
